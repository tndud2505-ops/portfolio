흩어진 작업과 아이디어를 실제로 만져볼 수 있는 도구로 남깁니다.

이곳은 이력서 요약보다 제가 직접 만든 프로젝트의 흐름을 보여주는 아카이브입니다. 원격 작업 요청, 여러 AI 작업 세션 관리, 중고거래 매물 비교, 주식 조건 검토, MCU 동작 학습처럼 반복해서 확인해야 하는 일을 화면과 기록이 남는 도구로 정리했습니다.

이 사이트에서 먼저 볼 것

문제, 화면, 동작 흐름이 남아 있는 프로젝트 원격 작업 관리, AI 작업 분업, 매물 비교, 투자 조건 검토, MCU 동작 학습을 각각 하나의 작업 기록으로 정리했습니다.

01

소개

복잡한 제어와 반복 작업을 이해 가능한 구조와 화면으로 바꿉니다.

저를 짧게 설명하면

자동차 임베디드 분야에서 모터 제어, 바디 제어기, AUTOSAR 자동화처럼 정확한 동작과 반복 검증이 중요한 업무를 경험했습니다. 개인 프로젝트에서는 이 경험을 바탕으로 상태를 놓치기 쉬운 작업, 반복 검색이 필요한 정보, 판단 기준이 섞이기 쉬운 흐름을 도구로 정리했습니다.

관찰한다 흐름을 나눈다 화면으로 확인한다 다시 볼 수 있게 남긴다

02

실무 경험

차량 제어, 기능 설계, 자동화 검증 경험을 실제 작업 중심으로 정리했습니다.

동아전기부품 · 모터 제어

차량 냉매 흐름을 제어하는 모터 구동 로직 개발

차량 내부 냉매 유로가 목표 위치까지 안정적으로 열리고 닫히도록 모터 제어 로직을 구현했습니다. PMSM 모터의 힘을 기어와 액추에이터로 전달하고, 위치 판단, 상전환, zero crossing 감지, sensorless 제어, STALL 감지, 재시도, 목표 위치 정지처럼 실제 동작 안정성에 영향을 주는 제어 흐름을 다뤘습니다.

LS Automotive · 바디 제어기

차량 바디 제어 기능 설계 및 선행 개발

운전자가 조작한 기능이 실제 차량 제어와 진단 흐름으로 이어지는 과정을 다뤘습니다. 조향 컬럼 모듈, 파워 시트 유닛, 통합 제어기 선행 과제를 통해 차량 기능이 사용자 입력, 통신, 진단, 기능안전 문서와 어떻게 연결되는지 경험했습니다. 구현뿐 아니라 문서 정리와 협업 산출물 관리까지 함께 다뤘습니다.

MOBASE ASEC · AUTOSAR 자동화

차량 소프트웨어 설정 자동화 및 검증 도구 개발

반복되는 차량 소프트웨어 설정 작업을 자동으로 코드화하고 검증하는 흐름을 만들었습니다. AUTOSAR ARXML을 읽고 C 코드 생성과 빌드 검증까지 이어지게 했으며, 생성 결과를 WSL/GCC 환경에서 확인할 수 있게 만드는 방향으로 작업했습니다.

Discord Remote Automation

Discord에 요청을 남기면 작업 상태와 결과를 원격으로 확인할 수 있는 관리 도구

로컬 PC 앞에 없어도 Discord에 작업 요청을 남기고, 진행 상황과 결과를 메시지로 확인할 수 있게 만든 도구입니다. 채널별 작업공간 연결, Codex 세션 이어가기, 승인 기반 실행, 로그 확인, 중지/재시작 흐름을 한곳에 묶어 원격에서도 작업 상태를 놓치지 않도록 했습니다.

  • Discord 채널과 로컬 프로젝트 매핑
  • Codex 작업 세션 재개와 상태 저장
  • 승인 기반 명령 실행과 중단
  • 로그·첨부파일·결과 메시지 회신
왜 만들었나
외부에 있을 때는 로컬 터미널을 계속 볼 수 없어 작업이 멈췄는지, 답변이 필요한지, 완료됐는지 늦게 알게 됩니다.
어떻게 만들었나
Discord 채널을 프로젝트 작업공간과 연결하고, 세션 저장·승인 버튼·작업 큐·로그 회신을 하나의 원격 제어 흐름으로 구성했습니다.
화면에서 볼 수 있는 것
사용자는 Discord에서 요청을 남기고, AI의 검토 결과와 작업 진행 상황을 메시지로 확인할 수 있습니다.

AI Work Coordination Board

큰 작업을 여러 AI 세션으로 나누고, 결과를 모아 검토하는 작업 보드

하나의 AI 세션에 모든 일을 맡기지 않고, 조사·구현·검증·리뷰를 나누어 진행하기 위한 작업 환경입니다. 각 세션의 중간 결과를 한곳에 모아 비교하고, 충돌이나 빠진 부분을 다시 확인할 수 있도록 구성했습니다. run, agent, issue, message, artifact 단위로 상태를 남기는 Control Tower MVP 성격의 실험 프로젝트입니다.

  • 역할별 AI 작업 세션 분리
  • run·issue·message 상태 기록
  • 작업 중단·재시도·재배정 흐름
  • 사람이 개입하는 승인 큐
왜 만들었나
큰 작업을 단일 에이전트로 처리하면 조사, 구현, 리뷰가 섞여 누락이나 판단 오류를 늦게 발견하기 쉽습니다.
어떻게 만들었나
요청을 역할별 세션으로 나누고, 각 세션의 상태·결정·산출물을 파일 기반으로 남겨 메인 화면에서 조율하는 구조로 설계했습니다.
화면에서 볼 수 있는 것
분석, 구현, 검증, 리뷰를 병렬로 확보하고 에이전트 간 피드백을 남겨 충돌·누락·품질 문제를 더 빨리 발견할 수 있습니다.

Used Market Tracking System

여러 중고거래 사이트의 매물을 한 화면에서 비교하는 도구

중고나라, 번개장터처럼 여러 사이트에 흩어진 관심 매물을 모아 가격, 등록 시점, 상태, 확인 포인트를 비교하는 도구입니다. collector, merge, scheduler, reporter 흐름을 나누고, 기준가 대비 편차·노이즈 필터·위험 신호·예상 수익을 계산해 자동 구매가 아니라 의사결정을 보조하는 화면으로 정리했습니다.

  • 사이트별 매물 수집과 정규화
  • 기준가 대비 편차와 후보 비교
  • 노이즈·중복·위험 신호 정리
  • Discord watch와 알림 흐름
왜 만들었나
사이트별로 검색 조건과 가격 기준이 흩어져 있어 좋은 매물을 놓치기 쉽습니다.
어떻게 만들었나
사이트별 데이터를 공통 스키마로 정리하고, 가격·상태·중복 여부·위험 신호를 비교할 수 있게 구성했습니다.
화면에서 볼 수 있는 것
반복 검색 시간을 줄이고, 후보별 가격·상태 차이를 한 화면에서 비교할 수 있습니다.

Stock Analysis & Auto-Trading Workbench

주식 조건을 검토하고 실행 기록을 분리해 보는 워크벤치

관심 종목, 차트 조건, 보유 상태, 주문 이력, 실행 전 체크 항목을 나누어 보여주는 개인용 검토 도구입니다. 매매를 대신 판단하는 화면이 아니라, 어떤 조건을 보고 어떤 상태에서 실행했는지 기록을 남기기 위해 구성했습니다. 실행 전에는 데이터 최신성, 매수 가능 시간, 손실 한도, 보유·미체결 주문 같은 preflight gate를 확인하도록 분리했습니다.

  • MA·ATR·거래량·RS 기반 후보 검토
  • 후보 종목·포지션·주문 이력 분리
  • 실행 전 리스크 게이트 확인
  • paper/real mode 상태 기록
왜 만들었나
조건 분석, 차트 해석, 매매 후보, 실행 상태가 섞이면 어떤 근거로 판단했는지 추적하기 어렵습니다.
어떻게 만들었나
조건 분석, 추천 점수, 매매 계획, 실행 상태를 분리하고, 실행 전 리스크 게이트를 통과한 항목만 기록하도록 구성했습니다.
화면에서 볼 수 있는 것
조건 근거, 보유 포지션, 주문 이력, 실행 상태를 함께 확인할 수 있어 검토 전후의 판단 흔적을 남길 수 있습니다.

Web-based MCU Simulator

코드를 실행하면 MCU 내부 상태가 어떻게 바뀌는지 보여주는 웹 학습 도구

브라우저에서 코드를 입력하고 실행하면 가상 보드, 레지스터 값, 전압·전류, 신호 흐름을 함께 확인할 수 있는 학습용 시뮬레이터입니다. Monaco 코드 편집기와 Web Worker 실행 환경을 분리하고, GPIO/ADC/PWM/UART/EXTI 레지스터 맵, 홀 센서 인터럽트와 모터 상태 변화를 tick 단위로 관찰할 수 있게 구성했습니다.

  • Monaco 기반 펌웨어 코드 입력
  • Web Worker 기반 가상 실행
  • GPIO·ADC·PWM·UART 상태 표시
  • Step/Run/Pause/Reset 관찰
왜 만들었나
실제 코드를 입력해도 레지스터, 전압·전류, 보드 상태 변화를 따로 확인해야 하면 MCU 실행 흐름을 이해하기 어렵습니다.
어떻게 만들었나
코드 입력, 빌드·실행 제어, 레지스터 맵, 가상 보드의 전압·전류와 신호 흐름을 브라우저 화면에 묶었습니다.
화면에서 볼 수 있는 것
별도 장비 없이도 코드 실행에 따른 레지스터와 신호 상태 변화를 웹 화면에서 반복 학습할 수 있습니다.
