02

실무 경험

차량 제어, 기능 설계, 자동화 검증 경험을 실제 작업 중심으로 정리했습니다.

동아전기부품 · 모터 제어

차량 냉매 흐름을 제어하는 모터 구동 로직 개발

차량 내부 냉매 유로가 목표 위치까지 안정적으로 열리고 닫히도록 모터 제어 로직을 구현했습니다. PMSM 모터의 힘을 기어와 액추에이터로 전달하고, 위치 판단, 상전환, zero crossing 감지, sensorless 제어, STALL 감지, 재시도, 목표 위치 정지처럼 실제 동작 안정성에 영향을 주는 제어 흐름을 다뤘습니다.

LS Automotive · 바디 제어기

차량 바디 제어 기능 설계 및 선행 개발

운전자가 조작한 기능이 실제 차량 제어와 진단 흐름으로 이어지는 과정을 다뤘습니다. 조향 컬럼 모듈, 파워 시트 유닛, 통합 제어기 선행 과제를 통해 차량 기능이 사용자 입력, 통신, 진단, 기능안전 문서와 어떻게 연결되는지 경험했습니다. 구현뿐 아니라 문서 정리와 협업 산출물 관리까지 함께 다뤘습니다.

MOBASE ASEC · AUTOSAR 자동화

차량 소프트웨어 설정 자동화 및 검증 도구 개발

반복되는 차량 소프트웨어 설정 작업을 자동으로 코드화하고 검증하는 흐름을 만들었습니다. AUTOSAR ARXML을 읽고 C 코드 생성과 빌드 검증까지 이어지게 했으며, 생성 결과를 WSL/GCC 환경에서 확인할 수 있게 만드는 방향으로 작업했습니다.

Remote Job Monitoring Console

장시간 실행 작업을 원격으로 모니터링하고 제어하는 운영 콘솔

장시간 실행되는 자동화 작업의 상태, 진행률, 예상 완료 시간, 로그, 중지/재시작 명령을 한 화면에서 확인하도록 만든 운영 패널입니다. 작업이 멈췄는지 직접 터미널을 보지 않아도 판단할 수 있고, 문제가 생기면 원격으로 바로 조치할 수 있게 구성했습니다.

  • 실행 상태와 진행률 추적
  • 잔여 시간과 실행 한도 표시
  • 중지/재시작 원격 제어
  • 로그와 작업 폴더 접근
해결한 문제
장시간 작업은 터미널을 계속 보지 않으면 멈춤, 오류, 완료 여부를 늦게 알게 됩니다.
구현한 방식
진행률, 남은 시간, 로그 보기, 폴더 열기, 원격 중지/재시작 버튼을 하나의 패널로 묶었습니다.
확인 가능한 결과
작업 확인 시간을 줄이고, 문제 발생 시 현장에 없어도 즉시 상태 확인과 중지/재시작이 가능합니다.

Multi-Agent Harness Engineering

여러 Codex 에이전트를 나누어 실행하고 조율하는 멀티 에이전트 하네스

큰 작업을 하나의 에이전트에게 맡기는 대신, 여러 AI 작업 세션을 역할별로 나누어 동시에 운용하는 Codex 작업 하네스입니다. 여러 작업 창을 나란히 두는 방식으로 각 에이전트가 조사, 구현, 검증, 리뷰 역할을 맡아 진행 상황을 공유하도록 구조를 설계했습니다.

  • 역할별 에이전트 분리 실행
  • tmux형 멀티 패널 작업 구조
  • 에이전트 간 결과 공유와 조율
  • 구현 후 리뷰 루프와 재검토 흐름
해결한 문제
큰 작업을 단일 에이전트로 처리하면 조사, 구현, 리뷰가 섞여 누락이나 판단 오류를 늦게 발견하기 쉽습니다.
구현한 방식
요구사항 분석, 코드 수정, 검증, 리뷰 역할을 분리하고 여러 에이전트가 각자의 결과를 공유·조율하는 하네스 구조로 설계했습니다.
확인 가능한 결과
여러 관점의 작업 결과를 병렬로 확보하고, 리뷰 루프를 통해 충돌·누락·품질 문제를 더 빨리 발견할 수 있습니다.

Used Market Tracking System

여러 중고거래 사이트의 매물을 수집하고 비교하는 통합 추적 시스템

중고나라, 번개장터 등 여러 사이트의 관심 키워드 매물을 모아 가격, 상태, 등록 시점, 확인 포인트를 비교하는 도구입니다. 반복 검색을 줄이고, 조건에 맞는 후보를 빠르게 추려볼 수 있도록 매물 정보를 한 화면에서 정리했습니다.

  • 키워드 기반 매물 수집
  • 가격 범위와 후보 비교
  • 중복 후보와 확인 포인트 정리
  • 관심 매물 빠른 발견
해결한 문제
사이트별로 검색 조건과 가격 기준이 흩어져 있어 좋은 매물을 놓치기 쉽습니다.
구현한 방식
키워드, 가격 범위, 후보 목록, 확인 포인트를 한 화면에 정리했습니다.
확인 가능한 결과
반복 검색 시간을 줄이고, 후보별 가격·상태 차이를 한 화면에서 비교할 수 있습니다.

Stock Analysis & Auto-Trading Workbench

주식 조건 분석과 자동매매 실행 흐름을 분리한 워크벤치

종목 후보, 조건 충족 여부, 자동매매 실행 전 체크 항목, 실행 상태와 기록을 분리해 보여주는 주식 자동매매 워크벤치입니다. 자동매매를 무작정 실행하는 화면이 아니라, 조건 검토와 실행 경계를 분리해 사람이 판단 흐름을 확인한 뒤 자동매매 상태를 관리할 수 있도록 구성했습니다.

  • 종목 후보와 조건 충족 여부 분리
  • 자동매매 실행 전 체크 단계
  • 매매 실행 상태와 결과 기록
  • 조건 검토와 자동 실행 경계 관리
해결한 문제
조건 분석, 매매 후보, 자동매매 실행 상태가 섞이면 어떤 근거로 실행됐는지 추적하기 어렵습니다.
구현한 방식
후보 목록, 조건 체크, 자동매매 실행 전 확인, 실행 상태, 기록 영역을 분리해 판단과 실행 흐름을 보이게 했습니다.
확인 가능한 결과
자동매매 실행 여부와 조건 근거를 함께 확인할 수 있어, 실행 전 검토와 실행 후 기록 추적이 쉬워집니다.

Web-based MCU Simulator

웹 기반 MCU 시뮬레이션 학습 도구

브라우저에서 가상 보드, 레지스터 값, 신호 흐름, 실행 제어를 함께 확인하는 MCU 학습 도구입니다. 실제 장비 없이도 코드 실행에 따라 보드 상태가 어떻게 바뀌는지 따라볼 수 있게 만들었습니다.

  • 보드 상태와 레지스터 표시
  • 신호 흐름과 제어 경로 시각화
  • 실행/정지 등 학습용 제어
  • 장비 없이 흐름 복습
해결한 문제
임베디드 학습은 코드, 레지스터, 보드 상태를 따로 확인해야 해서 실행 흐름을 이해하기 어렵습니다.
구현한 방식
가상 보드, 레지스터 표시, 신호 흐름, 실행 제어를 브라우저 화면에 묶었습니다.
확인 가능한 결과
장비 없이도 코드 실행에 따른 상태 변화를 한 화면에서 복습할 수 있습니다.

작업 방식

반복 작업을 분석하고, 흐름을 나누고, 검증 가능한 화면으로 구현합니다.

문제 정의

반복되는 불편을 먼저 찾습니다.

계속 확인해야 하는 상태, 놓치기 쉬운 조건, 수작업으로 반복되는 과정을 먼저 관찰합니다.

구조 설계

문제를 입력, 판단, 실행, 기록으로 나눕니다.

큰 기능을 한 번에 만들기보다 사용자가 확인해야 할 흐름과 기준을 단계별로 분리합니다.

구현·계측

설명을 실제 화면과 동작으로 옮깁니다.

상태, 로그, 버튼, 결과처럼 사용자가 직접 확인할 수 있는 형태로 구현합니다.

리뷰·검증

결과와 판단 근거를 함께 남깁니다.

왜 만들었는지, 어떤 기준으로 설계했는지, 다음에 무엇을 개선할 수 있는지 기록합니다.
