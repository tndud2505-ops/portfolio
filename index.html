01

동아전기부품

센서형 히트펌프 액추에이터

PMSM 모터와 기어를 통해 액추에이터를 움직이고, 냉매 유로를 바꿔 히트펌프 시스템의 온도 제어에 연결되는 구조를 다뤘습니다.

핵심은 단순 구동이 아니라 목표 위치에 정확히 도달하고 멈추는 것이었습니다. 상전환, 목표 위치 정지, STALL 감지, 재시도 로직을 직접 맡으며 제어 흐름 전체를 다뤘습니다.

제어 이론을 실제 제품 목적과 양산 대응으로 연결한 첫 실무 경험이었습니다. 냉매 유량 제어라는 시스템 목표와 펌웨어 구현을 하나로 묶어 본 경험입니다.

  • 목표 각도 기반 위치 제어
  • PMSM 상전환과 정지 타이밍
  • STALL 감지와 재시도
02

동아전기부품

센서리스 PT 액추에이터

센서 없이도 모터 상태를 읽어야 하는 구조에서 역기전력과 zero crossing 판단으로 액추에이터를 원하는 위치까지 제어하는 선행 과제를 수행했습니다.

초기 상전환, zero crossing 판단, sensorless 제어 유지, STALL 감지, 재시도와 목표 위치 정지까지 상태 흐름을 직접 다뤘습니다. 센서가 없을수록 로직의 밀도가 올라간다는 걸 실무로 겪었습니다.

단순 드라이버 디버깅이 아니라, 센서리스 제어에서 어떤 신호를 어떻게 해석해 상태 판단으로 이어가는지를 이해하고 구현한 경험이라는 점에서 중요한 프로젝트입니다.

  • zero crossing 판단
  • sensorless 위치 추정 흐름
  • 비정상 구간 보호와 복구 설계
03

LS Automotive

SCM과 차종별 양산 대응

조향 컬럼 위치 제어와 easy-exit, 메모리 위치 복원 같은 편의 기능이 실제 제어기 구조로 어떻게 연결되는지 다뤘습니다.

SCM은 기능 하나를 구현하는 문제보다 차종과 플랫폼에 따라 요구사항을 구조화하고 양산 단계 이슈를 디버깅하는 문제가 더 컸습니다. AUTOSAR Mobile Gene, MATLAB/Simulink, 모델 기반 설계 흐름을 실무에 붙여 본 경험입니다.

제어 대상이 커지면서 시스템 관점이 중요해지는 구간이었습니다. 차종별 대응, 양산 이슈, 실제 사용자 체감 품질이 모두 소프트웨어 구조와 연결되는 경험을 쌓았습니다.

  • SCM Platform 1.0 / 2.0 대응
  • 차종별 사양 해석과 구조화
  • 양산 단계 디버깅과 품질 대응
04

LS Automotive

PSU와 기능안전 문맥

Power Seat Unit에서는 시트 위치 제어 그 자체보다, 안전 요구사항과 진단·fail-safe 구조를 실제 개발 흐름에 넣는 경험이 중요했습니다.

ASIL-A 수준 요구사항을 반영하면서 디버깅, 기능 구현, 진단 및 fail-safe 로직, 그리고 SCD와 SDD 같은 산출물 작성을 함께 다뤘습니다. 코드와 문서가 같은 의미를 가져야 하는 프로젝트였습니다.

정상 동작을 만드는 것만으로는 충분하지 않았습니다. 어떤 고장을 진단 대상으로 볼지, 실패 시 어떤 안전 상태로 들어갈지, 그 판단을 문서와 구현 모두에서 설명 가능해야 했습니다.

  • 안전 요구사항 동시 반영
  • 진단 항목과 fail-safe
  • 문서와 구현의 정합성

LS Automotive

통합제어기 선행 구조화

시트, 램프, 카메라 모니터링과 여러 액추에이터 상태를 하나의 제어기 문맥 안에서 묶고 명령 흐름으로 연결하는 선행 과제에 참여했습니다.

이 프로젝트의 본질은 특정 기능 구현이 아니라 구조 정의였습니다. 여러 입력과 기능을 하나의 제어기 안에서 어떻게 나누고, 어떤 인터페이스와 이벤트 흐름으로 연결할지 검토하는 일이 중심이었습니다.

즉시 눈에 보이는 기능보다 제어기 아키텍처 감각을 키워준 경험이었습니다. 이후 양산 가능성을 고려해 구조를 먼저 정리하는 선행 개발의 무게를 이해하게 된 프로젝트입니다.

  • 다수 기능 통합 구조 검토
  • 인터페이스와 이벤트 흐름
  • 선행 단계 구조 검증
06

MOBASE ASEC

AUTOSAR 코드 생성과 검증 자동화

AUTOSAR ARXML을 해석해 일관된 C 코드를 생성하고, WSL/GCC 환경으로 검증까지 이어가는 자동화 흐름을 만들고 있습니다.

이 일은 단순 스크립트 작성보다 개발 구조를 바꾸는 작업에 가깝습니다. 어떤 ARXML 정보를 코드 구조로 바꿀지, 어떤 규칙을 자동화할지, 어떤 검증 단계를 필수로 둘지 판단해야 합니다.

자동차 임베디드 실무 경험이 개발 생산성과 품질 자동화로 확장되는 지점입니다. 반복 작업을 줄이면서도 결과를 더 검증 가능하게 만드는 방향을 꾸준히 밀고 있습니다.

  • ARXML 파싱과 생성 규칙 정리
  • Python 기반 코드 생성 툴링
  • WSL/GCC 기반 단위 검증 자동화

Operations workbench

KIWOM

추천 엔진과 자동매매 실행 엔진을 분리하고, readiness 체크와 운영 로그를 통해 실수성 주문을 줄이는 방향으로 설계한 워크벤치입니다.

이 프로젝트는 단순 자동매매 앱보다 운영 도구에 가깝습니다. 스캔, 추천, 종목 상세, 포지션 운영을 한 흐름으로 묶고, 실제 사용 중 무엇을 먼저 봐야 하는지 우선순위를 화면 구조로 정리했습니다.

핵심은 `잘 돌아간다`보다 `운영자가 판단 가능한 상태로 보인다`는 점이었습니다. 그래서 추천 결과뿐 아니라 readiness, 리스크, 포지션 상태를 같은 화면 안에서 보게 했습니다.

  • FastAPI + React 운영 구조
  • 추천 / 실행 엔진 분리
  • 운영 로그와 readiness 체크

Embedded education

Virtual MCU

브라우저 안에서 MCU 학습 환경을 구현해, 보드 상태와 레지스터 변화를 한 흐름 안에서 보이게 만든 교육형 시뮬레이터입니다.

기존 학습 자료가 문서와 코드 조각 중심이라면, 이 프로젝트는 학생이 입력과 출력, 레지스터 변화, 회로 흐름을 동시에 볼 수 있게 하는 데 초점을 뒀습니다.

보드, UART 입력, 핀 상태, 내부 로직을 하나의 학습면에 정리해서 단순 설명보다 실제 동작 구조를 이해하는 쪽에 더 가깝게 만들었습니다.

  • Web Worker 기반 시뮬레이션
  • 레지스터와 보드 상태 동시 관찰
  • 교육용 흐름 중심 인터페이스

Remote operations

Discord Agent

Discord 채널에서 로컬 Codex 세션을 제어하고 승인 흐름을 운영 도구처럼 다룰 수 있게 만든 원격 협업 컨트롤러입니다.

단순 봇이 아니라, 세션 상태와 승인, 사용량, 로그를 한 화면에서 보고 제어해야 하는 관리 도구 성격으로 접근했습니다.

명령 전달과 상태 확인이 섞여 있는 환경에서, 어떤 정보가 먼저 보여야 하는지와 어떤 동작이 한 번에 실행되어야 하는지를 인터페이스 구조로 정리한 프로젝트입니다.

  • 원격 세션 운영 흐름
  • 상태 / 승인 / 로그 통합
  • 협업형 컨트롤 패널

Browser automation

Site Auto

Chrome Extension 사이드패널을 중심으로 DOM snapshot, replay, repair, permission recovery를 한곳에서 다루게 만든 자동화 도구입니다.

자동화 프로젝트를 실제로 굴리다 보면 스크립트보다 운영 인터페이스가 더 중요해질 때가 많습니다. 이 프로젝트는 브라우저 안에서 상태를 확인하고 수동 개입과 자동 복구를 연결하는 데 초점을 뒀습니다.

즉 `자동화 코드`보다는 `자동화를 운영하는 패널`에 가깝게 설계했습니다.

  • Chrome Extension 사이드패널
  • Snapshot / Replay / Repair
  • 운영형 자동화 UI

Testing workbench

Site Security Check

Mouse replay와 isTrusted 검증을 묶어 브라우저 자동화가 실제 사용자 입력과 어디서 다르게 취급되는지 추적하는 워크벤치입니다.

자동화가 막히는 이유를 재현하려면 단순 성공/실패보다 이벤트 흐름과 브라우저 보안 문맥을 같이 봐야 했습니다. 그래서 실험용 스크립트가 아니라 비교 가능한 작업대처럼 구성했습니다.

결과적으로 이 프로젝트는 자동화 도구를 만드는 일과 자동화를 검증하는 일을 하나의 브라우저 실험 환경으로 묶은 사례입니다.

  • Mouse replay와 실사용 입력 비교
  • isTrusted 검증 흐름
  • Playwright 기반 워크벤치
