처음 보는 사람도 이해할 수 있게

직접 만든 도구를 쉽게 설명합니다.

복잡해 보이는 일을 차근차근 정리하고, 필요한 도구를 직접 만들어온 기록입니다. 실무 이력은 짧게 두고, 개인 프로젝트는 어떤 불편함을 줄이려고 만들었는지가 먼저 보이게 했습니다.

현재 초점

성실하게 관찰하고 직접 고쳐보기 작업 상태 확인, 단계별 일 정리, 중고 물건 비교, 주식 조건 점검, 웹 기반 학습 환경

01

한눈에 보는 소개

한 줄로 보면, 자동화와 제어를 실제 도구로 바꾸는 사람입니다.

핵심 소개

자동차 임베디드 실무에서 모터 제어, 바디 제어기, 기능안전, AUTOSAR 툴링을 다뤘고, 개인 프로젝트에서는 반복 작업을 줄이는 운영 도구와 감지 시스템을 만들어왔습니다. 관심사는 멋진 말보다 분명합니다. 사람이 계속 보고 있지 않아도 되는 구조를 만들고, 판단 기준을 화면과 로그로 남기는 것입니다.

사이트에서 보여줄 것

실무 경험은 “어떤 영역을 맡아봤는지”만 짧게 정리합니다. 실제 설득력은 개인 프로젝트에서 만듭니다. 각 프로젝트는 이름보다 먼저 문제, 만든 기능, 결과 화면, 다음에 확장할 수 있는 지점을 보여주는 방식으로 구성합니다.

02

과장 없이 정리한 이력

이력은 내용만 정리하고, 그림은 나중에 넣을 자리만 표시합니다.

동아전기부품 · 모터 제어

히트펌프 액추에이터 / PT 액추에이터 제어

PMSM 모터와 기어로 액추에이터를 움직이고, 냉매 유로 개폐와 목표 위치 정지까지 이어지는 제어 기능을 담당했습니다. 상전환, zero crossing 감지, sensorless 제어, STALL 감지, 재시도 로직처럼 실제 동작 안정성에 영향을 주는 판단을 직접 구현했습니다.

LS Automotive · 바디 제어기

SCM / PSU / 통합제어기 선행

조향 컬럼 모듈, 파워 시트 유닛, 통합 제어기 선행 과제를 통해 차량 기능이 사용감, 통신, 진단, 기능안전 문서와 어떻게 연결되는지 다뤘습니다. 구현뿐 아니라 양산 대응과 문서 정리까지 함께 경험했습니다.

MOBASE ASEC · AUTOSAR 툴링

ARXML 기반 코드 생성과 검증 자동화

AUTOSAR ARXML을 읽고 C 코드 생성과 검증까지 이어지는 자동화 흐름을 만들고 있습니다. 반복 설정을 줄이고, 생성 결과를 WSL/GCC 환경에서 확인할 수 있게 만드는 방향으로 작업하고 있습니다.

03

개인 프로젝트 기록

처음 보는 사람도 왜 만들었고 어디에 쓰는지 알 수 있게 정리했습니다.

작업 태도

제가 도구를 만들 때 중요하게 보는 것

자동화는 손을 놓기 위한 것이 아니라, 놓치기 쉬운 일을 줄이고 다시 확인할 수 있게 만들기 위한 방법이라고 생각합니다. 그래서 화면, 로그, 실행 전 확인 단계를 남기는 쪽을 중요하게 봅니다.

놓치지 않기 다시 볼 수 있게 남기기 사람이 덜 지치게 만들기

작업 확인 도구

작업 진행 상황을 한눈에 보는 알림 도구

컴퓨터에서 돌아가는 작업이 잘 진행되는지 멀리서도 확인할 수 있게 만든 도구입니다. 멈췄는지, 로그를 봐야 하는지, 다시 시작해야 하는지 같은 상태를 한 화면에서 확인할 수 있게 했습니다.

불편했던 점
작업 상태를 보려면 터미널이나 로그를 계속 확인해야 했습니다.
만든 해결
상태, 진행률, 중지/재시작, 로그 확인을 한 패널에 모았습니다.
  • 실행 상태, 진행률, 로그 확인을 한 화면에 배치
  • 멀리서도 작업 상태를 확인하고 필요한 조치를 할 수 있는 구조
  • 사람이 계속 화면을 보고 있지 않아도 되는 운영 방식 실험

작업 정리 방식

큰 일을 단계별로 나누고 확인하는 작업 관리판

큰 일을 한 번에 처리하지 않고, 계획, 구현, 검토, 마무리 단계로 나눠 진행하는 구조를 만들었습니다. 혼자 작업하더라도 여러 번 확인하며 실수를 줄이는 방식입니다.

불편했던 점
큰 작업은 중간 확인이 없으면 빠진 부분을 뒤늦게 발견하기 쉽습니다.
만든 해결
단계를 나누고, 검토 과정을 넣어 결과를 다시 합치는 흐름을 만들었습니다.
  • 작업을 역할별로 나누고 결과를 다시 합치는 방식 정리
  • 리뷰어 관점을 미리 세워 결과물을 여러 시선에서 점검
  • 반복 작업을 자동화 가능한 체크리스트로 변환
요청 분해 구현 리뷰 배포

생활형 비교 도구

중고 물건을 놓치지 않게 돕는 비교 알림 도구

중고나라와 번개장터 같은 곳에서 관심 있는 물건을 찾아보고, 가격과 상태를 비교할 수 있게 만든 도구입니다. 여러 사이트를 직접 돌아다니는 시간을 줄이는 것이 목적입니다.

불편했던 점
사이트마다 검색하고 가격과 상태를 따로 비교해야 했습니다.
만든 해결
관심 매물, 가격 범위, 확인 포인트를 한 화면에 정리했습니다.
  • 키워드 기반 매물 수집과 중복 후보 정리
  • 시세 범위와 관심 후보를 한 화면에서 비교
  • 놓치기 쉬운 매물을 빠르게 발견하기 위한 알림 구조

조건 점검 도구

주식 판단을 신중하게 돕는 조건 확인 도구

종목 추천, 조건 확인, 실행 전 점검을 단계별로 나눈 도구입니다. 혼자 알아서 거래하는 도구가 아니라, 사람이 최종 확인하기 전에 위험을 줄이기 위한 점검 화면을 목표로 만들었습니다.

불편했던 점
추천, 조건 확인, 실행 상태가 섞이면 판단이 흐려지고 위험해질 수 있습니다.
만든 해결
분석과 실행을 분리하고 실행 전 확인 단계와 기록을 먼저 보게 했습니다.
  • 분석과 실행 단계를 분리해 위험한 자동 실행을 줄임
  • 실행 전 확인 단계로 현재 상태를 확인
  • 결과와 로그를 남겨 다음 판단에 활용

학습 환경

웹 기반 MCU 시뮬레이션 환경

브라우저에서 MCU 학습 환경처럼 보드 상태와 레지스터 변화를 확인할 수 있게 만든 웹 프로젝트입니다. 복잡한 장비가 없어도 제어 흐름을 눈으로 따라가게 만드는 것이 목표였습니다.

불편했던 점
임베디드 학습은 보드 상태와 코드 실행 결과를 함께 보기 어렵습니다.
만든 해결
브라우저 안에서 보드, 레지스터, 실행 흐름을 같이 볼 수 있게 만들었습니다.
  • 웹에서 보드 상태, 레지스터, 실행 흐름을 시각화
  • 학습자가 “무엇이 바뀌었는지” 바로 볼 수 있는 인터페이스
  • 임베디드 개념을 웹 화면으로 옮기는 실험

04

보는 사람별 기준

보는 사람이 다르면, 좋아하는 포인트도 달라집니다.

면접관

직접 한 일이 어디까지인지 보이면 좋습니다.

각 프로젝트는 문제, 맡은 범위, 구현한 기능, 남은 결과가 분리되어 읽혀야 합니다.

일반인

전문 용어보다 “그래서 뭐가 편해졌는지”가 먼저 보여야 합니다.

중고 물건 비교, 작업 상태 확인, 실행 전 점검처럼 생활 속 행동으로 번역되는 문장을 앞에 둡니다.

프리랜서 고용인

맡기면 결과 화면까지 만들 사람인지 봅니다.

기능 설명만 하지 않고, 운영 화면, 체크 구조, 알림 구조처럼 실제 쓸 수 있는 형태를 강조합니다.

개발자 리뷰어

자동화가 어디서 시작되고 어디서 멈추는지 봅니다.

수집, 판단, 실행, 검증, 로그가 분리되어 있으면 단순 취미가 아니라 시스템으로 읽힙니다.
