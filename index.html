Personal projects

반복되는 문제를 구조화하고, 상태와 기록, 검증이 남는 도구로 바꿉니다.

배운 내용을 그냥 넘기지 않고, 실제로 써볼 수 있는 작은 도구로 만들어 봅니다. 원격 작업 관리, AI 작업 분업, 매물 추적, 조건 검토, MCU 시뮬레이터처럼 상태를 놓치기 쉬운 문제를 입력, 판단, 실행, 기록 흐름으로 나누고 실제 화면에서 확인 가능한 형태로 정리해 왔습니다.

01

소개

복잡하거나 반복되는 일을 보기 쉽게 정리하고, 직접 확인할 수 있게 만듭니다.

저를 짧게 설명하면

자동차 임베디드 분야에서 모터 제어, 바디 제어기, AUTOSAR 자동화처럼 정확한 동작과 반복 검증이 중요한 업무를 경험했습니다. 개인 프로젝트에서는 이 경험을 바탕으로 상태를 놓치기 쉬운 작업, 반복 검색이 필요한 정보, 판단 기준이 섞이기 쉬운 흐름을 작은 도구와 화면으로 정리했습니다.

관찰한다 흐름을 나눈다 화면으로 확인한다 다시 볼 수 있게 남긴다

02

실무 경험

차량 제어, 기능 설계, 자동화 검증 경험을 실제 작업 중심으로 정리했습니다.

동아전기부품 · 모터 제어

차량 내부 장치가 안정적으로 움직이도록 하는 모터 제어 개발

차량 내부 냉매 유로가 목표 위치까지 안정적으로 열리고 닫히도록 모터 구동 시퀀스와 위치 판단 로직을 구현·검증했습니다. PMSM 모터의 힘을 기어와 액추에이터로 전달하고, 상전환, zero crossing 감지, sensorless 제어, STALL 감지, 재시도, 목표 위치 정지처럼 실제 동작 안정성에 영향을 주는 판단 흐름을 맡았습니다.

LS Automotive · 바디 제어기

차량 바디 제어 기능 설계 및 선행 개발

운전자가 조작한 기능이 실제 차량 제어와 진단 흐름으로 이어지는 과정을 다뤘습니다. 조향 컬럼 모듈, 파워 시트 유닛, 통합 제어기 선행 과제를 통해 차량 기능이 사용자 입력, 통신, 진단, 기능안전 문서와 어떻게 연결되는지 경험했습니다. 구현뿐 아니라 문서 정리와 협업 산출물 관리까지 함께 다뤘습니다.

MOBASE ASEC · AUTOSAR 자동화

차량 소프트웨어 설정 자동화 및 검증 도구 개발

반복되는 차량 소프트웨어 설정 작업을 자동화하는 흐름을 구성했습니다. AUTOSAR ARXML 설정값을 추출해 C 코드 생성 입력으로 변환하고, 생성 결과를 WSL/GCC 빌드 환경에서 확인할 수 있게 만드는 방향으로 작업했습니다.

03

개인 프로젝트

공부로 끝내지 않고, 제가 실제로 써보려고 만든 프로젝트들을 모았습니다.

프로젝트 목록

밖에서도 작업 상태를 확인하는 도구

승인된 작업 요청의 상태, 로그, 결과를 Discord에서 확인하는 원격 작업 모니터링 도구

로컬 PC 앞에 없어도 Discord에 승인된 작업 요청을 남기고, 진행 상황과 결과를 메시지로 확인할 수 있게 만든 도구입니다. 채널별 작업공간 연결, 작업 큐, 세션 상태 저장, 로그 회신, 중지/재시작 흐름을 한곳에 묶어 원격에서도 작업이 멈췄는지, 답변이 필요한지, 완료됐는지 놓치지 않도록 했습니다.

  • Discord 채널과 로컬 프로젝트 매핑
  • Codex 작업 세션 재개와 상태 저장
  • 승인 기반 실행과 중단
  • 로그·첨부파일·결과 메시지 회신
문제 배경
로컬 PC 앞에 없어도 승인된 작업 요청의 진행 상태, 로그, 결과를 원격으로 확인할 수 있도록 만들었습니다.
동작 방식
Discord 채널을 로컬 프로젝트 작업공간과 연결하고, 승인 버튼, 작업 큐, 세션 실행, 로그 회신을 하나의 모니터링 흐름으로 구성했습니다.
구현 범위
Discord Bot API, 로컬 프로세스 제어, 작업 큐, 세션 상태 저장, 파일·로그 추적, 승인 기반 실행 구조를 구현했습니다.

AI에게 일을 나누어 맡기는 작업 구조

작업을 역할별 세션으로 분리하고, 산출물과 결정 이력을 한곳에서 비교하는 작업 보드

하나의 AI 세션에 모든 일을 맡기지 않고, 조사·구현·검증·리뷰를 역할별로 나누기 위한 작업 환경입니다. 각 세션의 중간 결과와 결정 이력을 한곳에 모아 비교하고, 충돌이나 빠진 부분을 다시 확인할 수 있도록 구성했습니다. 요청을 작업 단위로 쪼개고, 에이전트별 결과를 다시 합치는 흐름을 실험했습니다.

  • 역할별 AI 작업 세션 분리
  • 작업 번호, 문제, 대화 기록 정리
  • 작업 중단·재시도·재배정 흐름
  • 사람이 개입하는 승인 큐
문제 배경
큰 작업을 단일 에이전트로 처리하면 조사, 구현, 리뷰가 섞여 누락이나 판단 오류를 늦게 발견하기 쉽습니다.
동작 방식
요청을 역할별 세션으로 나누고, 각 세션의 상태·결정·산출물을 기록한 뒤 메인 보드에서 진행 상황과 충돌 지점을 통합 관리합니다.
구현 범위
작업 분해 설계, 세션 상태 모델링, 산출물 추적, 충돌 감지, 리뷰 흐름, 파일 기반 로그 관리를 구현했습니다.

중고거래 매물 추적

여러 중고거래 사이트의 후보를 수집·정리해 한 화면에서 비교하는 매물 추적 도구

중고나라, 번개장터처럼 여러 사이트에 흩어진 관심 매물을 모아 가격, 등록 시점, 상태, 확인 포인트를 비교하는 도구입니다. 허용된 범위의 검색 결과를 수집하고, 병합, 스케줄링, 알림 흐름을 나누어 기준가 대비 편차·중복 제거·주의 신호·확인 우선순위를 계산했습니다. 자동 구매가 아니라 관심 후보를 빠르게 비교하는 화면으로 정리했습니다.

  • 사이트마다 다른 매물 정보 정리
  • 기준가 대비 편차와 후보 비교
  • 중복되거나 관련 없는 매물 제외
  • 주의해서 볼 항목과 알림 흐름
문제 배경
여러 사이트에 흩어진 매물을 반복 검색하고 가격·상태·등록 시점을 직접 비교하는 시간을 줄이기 위해 만들었습니다.
동작 방식
사이트별 검색 결과를 공통 데이터 구조로 정리하고, 키워드·가격 범위·상태·중복 여부로 필터링한 뒤 관심 조건에 맞는 매물을 축적하고 알림으로 전달합니다.
구현 범위
검색 조건 관리, 데이터 수집·정리, 중복 제거, 가격 이력 저장, 필터링 로직, 스케줄러, Discord 알림 연동을 구현했습니다.

주식 조건 검토 도구

주식 조건과 실행 기록을 따로 확인하는 개인용 검토 화면

관심 종목, 차트 조건, 보유 상태, 주문 이력, 실행 전 체크 항목을 나누어 보여주는 개인용 검토 도구입니다. 매매를 대신 판단하는 화면이 아니라, 어떤 조건을 보고 어떤 상태에서 실행했는지 기록을 남기기 위해 구성했습니다. 실행 전에는 데이터 최신성, 매수 가능 시간, 손실 한도, 보유·미체결 주문 같은 안전 확인 단계를 분리했습니다.

  • MA·ATR·거래량·RS 기반 후보 검토
  • 후보 종목·포지션·주문 이력 분리
  • 실행 전 안전 확인 단계
  • 모의/실행 모드 상태 기록
문제 배경
조건 분석, 차트 해석, 후보 종목, 주문 상태가 섞이면 어떤 근거로 판단했는지 추적하기 어렵기 때문에 실행 전 검토 흐름을 분리했습니다.
동작 방식
조건 검색, 규칙 기반 후보 점검 결과, 실행 계획, 보유 포지션, 주문 이력을 분리하고, 실행 전 데이터 최신성·매수 가능 시간·안전 조건을 확인하도록 구성했습니다.
구현 범위
주식 API 연동, 조건식 평가, 시세 데이터 처리, 주문 상태 추적, 모의/실행 모드 분리, 실행 전 안전 확인 단계를 구현했습니다.

장비 없이 흐름을 확인하는 MCU 학습 도구

코드를 실행하면 MCU 내부 상태가 어떻게 바뀌는지 보여주는 웹 학습 도구

브라우저에서 코드를 입력하고 실행하면 가상 보드, 레지스터 상태, 입력값, 주변장치 신호 흐름을 함께 확인할 수 있는 학습용 시뮬레이터입니다. Monaco 코드 편집기와 Web Worker 실행 환경을 분리하고, GPIO/ADC/PWM/UART/EXTI 레지스터 맵, 홀 센서 인터럽트와 모터 상태 변화를 tick 단위로 관찰할 수 있게 구성했습니다.

  • Monaco 기반 펌웨어 코드 입력
  • Web Worker 기반 가상 실행
  • GPIO·ADC·PWM·UART 상태 표시
  • Step/Run/Pause/Reset 관찰
문제 배경
실제 장비 없이도 코드 실행에 따른 레지스터, GPIO, ADC, PWM, UART 상태 변화를 한 화면에서 학습할 수 있도록 만들었습니다.
동작 방식
브라우저에서 코드를 입력하고 Web Worker로 가상 실행을 분리한 뒤, 실행 단계에 따라 레지스터와 주변장치 신호 변화를 시각적으로 업데이트합니다.
구현 범위
Monaco Editor, Web Worker, 가상 MCU 상태 모델, GPIO·ADC·PWM·UART 시뮬레이션, Step/Run/Pause 제어 로직을 구현했습니다.

작업 방식

반복 작업을 분석하고, 흐름을 나누고, 다시 확인할 수 있는 화면으로 구현합니다.

문제 정의

반복되는 불편을 먼저 찾습니다.

계속 확인해야 하는 상태, 놓치기 쉬운 조건, 수작업으로 반복되는 과정을 먼저 관찰합니다.

구조 설계

문제를 입력, 판단, 실행, 기록으로 나눕니다.

큰 기능을 한 번에 만들기보다 사용자가 확인해야 할 흐름과 기준을 단계별로 분리합니다.

구현·계측

설명을 실제 화면과 동작으로 옮깁니다.

상태, 로그, 버튼, 결과처럼 사용자가 직접 확인할 수 있는 형태로 구현합니다.

리뷰·검증

결과와 판단 근거를 함께 남깁니다.

왜 만들었는지, 어떤 기준으로 설계했는지, 다음에 무엇을 개선할 수 있는지 기록합니다.
