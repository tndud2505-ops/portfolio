과장 없이 정리한 이력

동아전기부품 · 모터 제어

차량 냉매 유로를 여닫는 모터 제어 개발

차량 내부 냉매 흐름을 정해진 위치까지 안정적으로 열고 닫는 제어 로직을 구현했습니다. PMSM 모터의 힘을 기어와 액추에이터로 전달하고, 상전환, zero crossing 감지, sensorless 제어, STALL 감지, 재시도, 목표 위치 정지처럼 실제 동작 안정성에 영향을 주는 판단 로직을 직접 구현했습니다.

LS Automotive · 바디 제어기

차량 바디 제어 기능 설계 및 선행 개발

운전자가 조작한 기능이 실제 차량 제어와 진단 흐름으로 이어지는 과정을 다뤘습니다. 조향 컬럼 모듈, 파워 시트 유닛, 통합 제어기 선행 과제를 통해 차량 기능이 사용자 입력, 통신, 진단, 기능안전 문서와 어떻게 연결되는지 경험했습니다. 구현뿐 아니라 문서 정리와 협업 산출물 관리까지 함께 다뤘습니다.

MOBASE ASEC · AUTOSAR 자동화

차량 소프트웨어 설정을 코드로 바꾸는 자동화 도구

반복되는 차량 소프트웨어 설정 작업을 자동으로 코드화하고 검증하는 흐름을 만들었습니다. AUTOSAR ARXML을 읽고 C 코드 생성과 빌드 검증까지 이어지게 했으며, 생성 결과를 WSL/GCC 환경에서 확인할 수 있게 만드는 방향으로 작업했습니다.

Remote Job Monitoring Console

장시간 실행 작업을 원격으로 모니터링하고 제어하는 도구

장시간 실행되는 자동화 작업의 상태, 진행률, 예상 완료 시간, 로그, 중지/재시작 명령을 한 화면에서 확인하도록 만든 운영 패널입니다. 작업이 멈췄는지 직접 터미널을 보지 않아도 판단할 수 있게 했습니다.

  • 실행 상태와 진행률 추적
  • 잔여 시간과 한도 표시
  • 중지/재시작 원격 제어
  • 로그와 작업 폴더 접근
해결한 문제
장시간 작업은 터미널을 계속 보지 않으면 멈춤, 오류, 완료 여부를 늦게 알게 됩니다.
구현한 방식
진행률, 남은 시간, 로그 보기, 폴더 열기, 원격 중지/재시작 버튼을 하나의 패널로 묶었습니다.
확인 가능한 결과
작업 확인 시간을 줄이고, 문제 발생 시 현장에 없어도 즉시 중지/재시작할 수 있습니다.

Task Decomposition Workflow

Codex 작업을 요청·구현·리뷰 단계로 나누는 워크플로 보드

Codex로 큰 작업을 진행할 때 요청, 분해, 구현, 리뷰, 배포 단계를 분리해 누락을 줄이는 작업 보드입니다. 각 단계마다 담당 역할과 확인 기준을 남겨, 작업자와 리뷰어가 같은 기준으로 진행 상태를 확인하게 했습니다.

  • 요구사항과 제약 분리
  • 작업 단위와 담당 역할 정의
  • 리뷰 관점과 검증 기준 기록
  • 배포 후 확인 항목 관리
해결한 문제
요청을 바로 구현하면 범위 누락, 리뷰 기준 불일치, 배포 확인 누락이 뒤늦게 발견됩니다.
구현한 방식
요청 정의, 작업 분해, 구현 체크, 리뷰 기준, 배포 확인을 순서대로 기록하는 보드를 만들었습니다.
확인 가능한 결과
현재 단계, 다음 확인 지점, 리뷰 기준을 한눈에 보여 작업 누락과 재작업을 줄입니다.
요청 목표와 제약 정리
분해 작업 단위와 담당 역할 분리
구현 파일 수정과 중간 검증
리뷰 누락, 오류, 표현 품질 점검
배포 공개 페이지 반영 확인

Used Market Tracking System

여러 중고거래 사이트의 매물을 모아 비교하는 추적 도구

중고나라, 번개장터 등 여러 사이트의 관심 키워드 매물을 모아 가격, 상태, 등록 시점, 확인 포인트를 비교하는 도구입니다. 반복 검색을 줄이고, 조건에 맞는 후보를 빠르게 추려보는 데 초점을 맞췄습니다.

  • 키워드 기반 매물 수집
  • 가격 범위와 후보 비교
  • 중복 후보와 확인 포인트 정리
  • 관심 매물 빠른 발견
해결한 문제
사이트별로 검색 조건과 가격 기준이 흩어져 있어 좋은 매물을 놓치기 쉽습니다.
구현한 방식
키워드, 가격 범위, 후보 목록, 확인 포인트를 한 화면에 정리했습니다.
확인 가능한 결과
반복 검색 시간을 줄이고, 후보별 가격·상태 차이를 한 화면에서 비교할 수 있습니다.

Stock Condition Review Dashboard

매매 판단 전 조건과 근거를 분리해 확인하는 검토 대시보드

종목 후보, 조건 충족 여부, 실행 전 체크 항목, 기록을 분리해 보여주는 검토용 화면입니다. 자동으로 매수·매도하는 도구가 아니라, 사람이 최종 판단하기 전에 조건과 근거를 빠뜨리지 않도록 만든 안전 점검 도구입니다.

  • 추천 후보와 조건 분리
  • 실행 전 확인 단계 표시
  • 판단 근거와 결과 기록
  • 자동 실행 위험 경계 분리
해결한 문제
추천 결과, 조건 충족 여부, 실행 상태가 섞이면 근거 없이 판단하기 쉽습니다.
구현한 방식
후보 목록, 조건 체크, 실행 전 확인, 기록 영역을 분리해 판단 흐름을 보이게 했습니다.
확인 가능한 결과
매매 실행보다 확인 절차를 앞에 두어 사람이 마지막 판단을 하도록 안전 경계를 만들었습니다.

Web-based MCU Simulator

웹 기반 MCU 시뮬레이션 학습 도구

브라우저에서 가상 보드, 레지스터 값, 신호 흐름, 실행 제어를 함께 확인하는 MCU 학습 도구입니다. 실제 장비 없이도 코드 실행에 따라 보드 상태가 어떻게 바뀌는지 따라볼 수 있게 만들었습니다.

  • 보드 상태와 레지스터 표시
  • 신호 흐름과 제어 경로 시각화
  • 실행/정지 등 학습용 제어
  • 장비 없이 흐름 복습
해결한 문제
임베디드 학습은 코드, 레지스터, 보드 상태를 따로 확인해야 해서 실행 흐름을 이해하기 어렵습니다.
구현한 방식
가상 보드, 레지스터 표시, 신호 흐름, 실행 제어를 브라우저 화면에 묶었습니다.
확인 가능한 결과
장비 없이도 코드 실행에 따른 상태 변화를 한 화면에서 복습할 수 있습니다.

04

작업을 정리하는 방식

문제를 발견하면 작게 나누고, 확인 가능한 화면으로 남깁니다.

관찰

먼저 불편한 지점을 정확히 봅니다.

반복해서 확인하는 일, 놓치기 쉬운 일, 손으로 계속 해야 하는 일을 먼저 찾습니다.

분해

큰 흐름을 작은 단계로 나눕니다.

한 번에 해결하려 하지 않고 입력, 판단, 실행, 기록으로 나눠서 구조를 잡습니다.

구현

생각을 실제로 눌러볼 수 있는 화면으로 만듭니다.

기능 설명에서 멈추지 않고 상태, 로그, 버튼, 결과가 보이는 형태까지 옮깁니다.

기록

나중에 다시 볼 수 있게 남깁니다.

왜 만들었는지, 어떤 판단을 했는지, 다음에 무엇을 고칠 수 있는지 같이 정리합니다.
