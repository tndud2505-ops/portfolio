# 김수영 채널별 업로드 전략

## 목표

하나의 마스터 프로필을 바탕으로 `사람인`, `잡코리아`, `리멤버`, `LinkedIn`, `개인 사이트`, `네이버 블로그`에 맞는 톤과 길이로 재가공해 업로드 부담을 최소화합니다.

핵심 포지셔닝은 아래 한 줄로 통일합니다.

`자동차 임베디드 SW 실무를 기반으로 AUTOSAR, MBD, 기능안전, 모터 제어를 수행해 왔고, 최근에는 Python 기반 자동화와 검증 툴링까지 확장하고 있는 엔지니어`

## 업로드 우선순위

1. 개인 사이트 공개
2. LinkedIn `About / Experience / Featured` 입력
3. 사람인 `이력서 + 경력기술서 + 포트폴리오 URL` 등록
4. 리멤버 `헤드라인 + 요약 경력 + 핵심 키워드` 입력
5. 잡코리아 `경력 + 희망 직무 + 스킬 + 자기소개서 + 포트폴리오` 정리
6. 네이버 블로그에 프로젝트 글 순차 업로드

## 플랫폼별 전략

### 1. 사람인

사람인은 검색형 채용 플랫폼이라서 `직무 키워드`, `경력기술서`, `포트폴리오 첨부`, `자기소개서`까지 구조적으로 채워두는 편이 유리합니다.

입력 전략:

- 희망 직무를 넓게 쓰지 말고 `자동차 임베디드 SW`, `AUTOSAR`, `제어기 SW`, `검증 자동화`, `모터 제어`로 쪼개서 반영
- 경력기술서는 회사별로 `무엇을 개발했는지`, `어떤 문제를 해결했는지`, `무엇을 양산/검증했는지`를 bullet 형태로 작성
- 포트폴리오 문서 파일과 개인 사이트 URL을 함께 등록
- 자기소개서는 성장 배경보다 `실무 강점`, `도메인 이해`, `자동화 확장성` 중심으로 작성

반영 포인트:

- 검색용 키워드: `C`, `Python`, `AUTOSAR`, `MATLAB/Simulink`, `기능안전`, `SCM`, `PSU`, `BLDC`, `PMSM`, `검증 자동화`
- 실무 프로젝트는 차종/세대/양산 대응 경험을 앞에 배치
- 개인 프로젝트는 `현재도 만들고 있는 기술 확장성`의 근거로 사용

### 2. 리멤버

리멤버는 스카웃형 플랫폼 성격이 강하므로 긴 서술보다 `짧은 헤드라인`, `압축된 경력 요약`, `검색되는 키워드`가 더 중요합니다.

입력 전략:

- 첫 줄에서 `자동차 임베디드 SW`와 `AUTOSAR`, `모터제어`, `검증자동화`를 바로 노출
- 소개문은 3~5문장으로 압축
- 회사별 역할은 길게 설명하지 말고 `도메인 + 담당 범위 + 핵심 강점` 중심으로 요약
- 관심 포지션을 명확하게 적어 스카웃 검색 적합도를 높임

반영 포인트:

- `양산 대응`, `MCU 변경 재설계`, `AUTOSAR`, `기능안전`, `Python 자동화` 같이 스카웃 검색에 걸리기 쉬운 단어를 전면 배치
- 현재 직무와 확장 가능 직무를 함께 명시
- 긴 프로젝트 설명은 빼고, 사이트 링크나 LinkedIn으로 확장시키는 구조 사용

### 3. 잡코리아

잡코리아는 공식 FAQ 기준으로 `경력`, `희망 직무`, `희망 근무지`가 기본 축이고, 선택 항목으로 `스킬`, `자기소개서`, `포트폴리오`, `자격증`, `어학`, `수상`, `교육`, `인턴·대외활동`, `해외 경험`을 채울 수 있습니다.

입력 전략:

- 간편 프로필은 `경력`, `희망 직무와 스킬`, `희망 근무지` 중심으로 먼저 완성
- 기본 이력서에는 `경력`, `희망 직무`, `희망 근무지`를 정확히 넣고, 선택 항목은 가능한 많이 채움
- 스킬 항목은 검색되는 기술명 위주로 정리
- 포트폴리오에는 문서 파일과 사이트 URL 연결성을 맞춤
- 자기소개서는 성장 배경보다 실무 강점과 문제 해결 방식을 중심으로 작성

반영 포인트:

- 공식 확인된 항목 구조에 맞춰 실제 입력 순서와 문구를 분리
- 실무 프로젝트 중심으로 쓰고 개인 프로젝트는 포트폴리오/사이트에서 확장
- 스킬, 자기소개서, 포트폴리오를 함께 채워 완성도를 높임

### 4. LinkedIn

LinkedIn은 `Headline`, `About`, `Experience`, `Featured`, `Projects`, `Skills`를 함께 다듬어야 합니다. 영문 검색 노출을 고려해 영어 중심으로 작성하되, 한국 경력 맥락이 흐려지지 않게 구성합니다.

입력 전략:

- Headline은 직무명과 핵심 기술을 한 줄에 배치
- About은 `현재까지의 경력 축`, `강점`, `관심 역할` 순서로 3단락 구성
- Experience는 회사별로 2~4개 bullet로 구조화
- Featured에는 개인 사이트, 대표 프로젝트 링크, 포트폴리오 PDF를 우선 등록
- Skills는 검색되는 키워드 위주로 정리

반영 포인트:

- `Automotive Embedded Software Engineer`, `AUTOSAR`, `Motor Control`, `Validation Automation`을 반복 노출
- 개인 프로젝트는 `tooling`, `automation`, `learning platform`, `operational product` 성격으로 분류
- 한국어 원문은 개인 사이트나 PDF에서 보완하고, LinkedIn은 읽기 쉬운 영어로 압축

### 5. 개인 사이트

개인 사이트는 가장 깊이 있는 설명 공간입니다. 이력서의 요약본이 아니라 `브랜드 아카이브`처럼 보여야 합니다.

입력 전략:

- Hero에서 정체성을 한 번에 설명
- 실무 프로젝트는 `배경 - 내가 한 일 - 결과 - 의미` 구조로 정리
- 개인 프로젝트는 실제 캡처와 구조 설명을 함께 배치
- 현재 공개 기준에서는 실무 프로젝트와 근거가 강한 개인 프로젝트 위주로 운영

반영 포인트:

- 추상적인 3D 이미지는 쓰지 않고 실제 캡처와 이해 중심 도해만 사용
- BLDC, AUTOSAR, KIWOM 같은 핵심 주제는 설명 그림을 같이 붙여 이해를 돕기
- LinkedIn, 사람인, 리멤버로 넘어갈 수 있는 링크 허브 역할도 수행

### 6. 네이버 블로그

블로그는 검색 유입과 장문 설명, 업데이트 이력을 남기는 공간으로 활용합니다.

입력 전략:

- 한 글에 한 프로젝트 또는 한 경력 축만 다루기
- 제목은 검색형으로 작성
- 구조는 `배경 - 문제 - 구현 - 결과 - 배운 점`으로 통일

반영 포인트:

- 개인 사이트보다 문장 길이를 늘리고, 과정과 시행착오를 더 적음
- 대학 프로젝트는 제외하고, 실무/현재 프로젝트만 직접 업로드용 패키지로 운영
- 프로젝트 추가 시 블로그부터 공개하고 사이트에는 요약을 반영하는 순서로 운영 가능

## 최소 작업 업로드 루틴

1. `채널별_복붙_원고.md`에서 해당 플랫폼 문구 복사
2. `플랫폼별_입력_패키지.md`에서 필드별 입력 순서 확인
3. 사람인에는 DOCX/PDF와 사이트 URL 동시 등록
4. LinkedIn에는 사이트 URL과 대표 프로젝트를 `Featured`에 연결
5. 리멤버는 압축형 소개 우선 입력, 잡코리아는 공식 이력서 항목을 최대한 많이 채움

## 유지보수 규칙

- 경력 변경 시 `프로젝트_마스터_원고.md`를 먼저 수정
- 새로운 개인 프로젝트가 생기면 `개인 사이트 -> LinkedIn Featured -> 블로그` 순으로 추가
- 사람인, 잡코리아, 리멤버는 분기별 1회 이상 키워드와 헤드라인 점검
- LinkedIn은 새 프로젝트 공개 시점마다 `Featured`와 `Skills`를 함께 갱신

## 참고 메모

- 사람인은 이력서 작성 시 `포트폴리오 및 기타문서`, `경력기술서`, `자기소개서` 같은 추가 항목을 둘 수 있고, 파일 또는 URL 이력서 등록도 지원합니다.
- 잡코리아는 공식 FAQ 기준으로 필수 항목과 선택 항목이 구분되어 있으며, 선택 항목으로 `스킬`, `자기소개서`, `포트폴리오` 등을 채울 수 있습니다.
- LinkedIn은 `About`, `Position`, `Skills`, `Featured`, `Projects` 같은 섹션을 직접 추가할 수 있습니다.
- 리멤버는 공개 보도 기준으로 프로필에 기본 경력만 입력해도 스카웃 제안을 받을 확률이 높아지는 구조이므로, 압축된 경력 정보 완성도가 특히 중요합니다. 다만 개인 프로필 입력 필드 자체는 공식 도움말 수준으로 충분히 검증된 상태가 아니므로 운영 전략 성격이 일부 포함됩니다.
